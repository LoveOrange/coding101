# [WIP] Lec 1: MapReduce

## 1. 背景介绍

MapReduce 是 Google 于 2004 年提出的分布式计算模型，用于处理大规模数据集的并行处理。

MapReduce 的核心思想是将单机无法处理的大规模的计算任务拆分为多个子任务，并行处理每一个自任务，最后将结果合并。

MapReduce 的论文地址为：[MapReduce: Simplified Data Processing on Large Clusters](https://pdos.csail.mit.edu/6.824/papers/mapreduce.pdf)。

需要注意，本文为 MapReduce 的学习笔记，会记录 MapReduce 模型上的重要概念，对于工程化的细节不会过多涉及，有需要可以参考论文原文。

## 2. Google 面对的挑战

作为世界上最大的搜索引擎，即使早在 2004 年，Google 需要处理的数据量也是以 PB 为单位。即使抛开硬件成本，这些数据也无法放到单机中进行处理。

如何能够高效的处理这些数据，是 Google 所面对的挑战。

一种很自然的解决方案，就是将原始数据进行分片，当分片足够小后，就可以在消费级的电脑上处理每一个分片，最后将所有分别的结果合并，即可获得最终的结果。

那么接下来的问题，变成了：

1. 如何对原始数据进行分片，如何处理每一个分片
2. 如何在集群中处理每一个分片任务，如何保证每一个任务可以正确执行
3. 如何保证任务的执行效率
4. 如何降低用户的使用成本，用户无需了解分布式系统的处理细节

## 3. MapReduce 编程模型

对于用户来说，给定一组输入数据，用户需要提供 Map 与 Reduce 函数。

Map 函数：

1. 输入：(Key, Value) 键值对
2. 作用：将输入的键值对映射为一组中间结果
3. 输出：(Key, Value[]) 键值对。经过 Map 处理过的原始数据，可能有多个 Value 对应同一个 Key。

Reduce 函数：

1. 输入：(Key, Value[]) 键值对
2. 作用：将输入的键值对映射为一组最终结果
3. 输出：(Key, Value) 键值对
