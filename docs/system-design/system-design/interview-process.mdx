# 系统设计面试流程

系统设计面试通常是一个开放式的对话，而你，作为面试者，被期待着**主导**这场对话。

系统面试的流程通常包括一下步骤。

## 1. 描述使用场景、约束和假设

首先需要讨论清楚问题的背景。理清楚系统的所有需求与问题，不断地与面试官交流，不停地提问，保证我们清楚地理解了系统的需求与限制条件，与面试官讨论可能的一些假设条件。

在这一环节，需要讨论清楚以下几个问题：

1. 用户侧的需求
    1. 面向的用户是什么
    2. 有多少用户
    3. 用户会怎么使用它
2. 系统的功能
    1. 系统的作用是什么
    2. 系统的输入输出是什么
3. 系统的规模
    1. 期望存储的数据规模是什么
    2. 系统的请求量是多少
    3. 系统的读写比例是多少

## 2. 高层设计

使用核心的组件来描绘一个高层次的设计。这个设计应该是一个简单的框架，用来解决问题的核心部分。

1. 列出所有的核心组件，将组件按照数据的流向连接起来
2. 证明这个设计可以满足需求

## 3. 细节设计

对每一个核心组件进行深入的分析。这部分根据不同的题目有不同的实现，会在具体的问题中进行详细的解释。

## 4. 扩展设计

要针对不同的假设条件，讨论系统的扩展性。这部分通常包括：

1. 负载均衡
2. 水平扩展
3. 缓存
4. 数据库分片
